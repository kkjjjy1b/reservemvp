# UI Invariants

## Login

- Preserve the current two-panel login layout and copy hierarchy by default.
- Keep the login form behavior and error handling semantics stable.

## Main Timeline

- Preserve the day-based timeline workflow.
- Preserve room ordering and reservation block readability.
- Keep empty-slot create flow and reservation-detail flow intuitive and quick.

## Account Page

- Preserve self-service profile editing and reservation history access.

## General Rule

- If the user asks for a bug fix, do not widen the scope into a redesign.
