# Push Journal

각 push 단위로 목표, 진행한 내용, 남은 일, 작업 원칙을 기록하는 문서다.

## 기록 원칙
- push가 끝날 때마다 최신 항목을 맨 위에 추가한다.
- 한 항목에는 최소한 `목표`, `진행한 것`, `남은 것`, `꼭 지켜야 하는 원칙`을 남긴다.
- 세부 구현을 모두 나열하기보다, 다음 환경에서 바로 이어서 일할 수 있는 수준으로 압축한다.
- 제품 정책/DB/API 계약 자체가 바뀌면 이 문서만이 아니라 `docs/change-log.md`에도 함께 반영한다.

## 2026-03-21 - origin/main `77cc45b`
### 목표
- 예약을 개인 단위 이벤트에서 사람 중심 협업 흐름으로 확장한다.
- 사용자 프로필과 참여자 개념을 도입하되, 기존 예약 권한 모델은 유지한다.
- 운영 시 필요한 후속 배포 조건까지 함께 정리한다.

### 진행한 것
- 사용자 프로필 아바타 업로드/표시 흐름을 추가했다.
- 예약에 참여자 구조를 도입하고, 검색 API와 입력 UI를 연결했다.
- 예약 생성/수정/상세 조회에서 참여자 정보를 다루도록 서버와 UI를 확장했다.
- 예약 생성 시 참여자 메일 알림을 보낼 수 있도록 메일 유틸을 추가했다.
- Prisma schema, migration, seed를 새 구조에 맞게 갱신했다.
- 실제 런타임과 맞지 않던 DB `sessions` 테이블을 제거하는 방향으로 정리했다.
- 설치 후 Prisma Client가 바로 생성되도록 `postinstall`에 `prisma generate`를 연결했다.

### 남은 것
- 운영 환경에 `BLOB_READ_WRITE_TOKEN`, `RESEND_API_KEY`, `RESEND_FROM_EMAIL`, `APP_URL` 적용 여부를 확정해야 한다.
- 운영 DB에 최신 migration 적용 여부를 확인해야 한다.
- 프로필 이미지 업로드 UX와 기본 아바타 스타일을 운영 기준으로 확정해야 한다.
- 참여자 메일 발송을 실제 운영 환경에서 검수해야 한다.
- 다음 작업에서도 `내 예약` 기준이 참여자가 아니라 소유자 기준으로 유지되는지 계속 확인해야 한다.

### 꼭 지켜야 하는 원칙
- 예약 소유자와 참여자를 명확히 분리한다.
- `내 예약`은 소유 예약만 보여준다.
- 메일/이미지 업로드 같은 부가 기능이 실패해도 핵심 예약 흐름은 최대한 유지한다.
- 사람 식별성 개선은 유지하되, 예약 생성과 조회 흐름을 더 무겁게 만들지 않는다.
- 세션/인증 구조는 현재의 stateless signed cookie 기준과 어긋나지 않게 유지한다.
