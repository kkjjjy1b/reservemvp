# Context Primer

새 환경이나 새 스레드에서 이 프로젝트를 이어받을 때 가장 먼저 읽는 문서다.

## 먼저 읽을 순서
1. 이 문서에서 현재 방향과 작업 원칙을 파악한다.
2. `docs/push-journal.md`에서 가장 최근 push의 목표, 진행 내용, 남은 일을 확인한다.
3. `docs/change-log.md`에서 제품/DB/API 수준의 중요한 결정만 확인한다.
4. 필요하면 `docs/spec.md`, `docs/requirements.md`로 세부 정책을 확인한다.

## 현재 제품 방향
- 이 프로젝트는 사내 회의실 예약 MVP다.
- 핵심 경험은 일간 타임라인에서 빠르게 현황을 보고, 빈 슬롯에서 바로 예약을 만드는 것이다.
- 최근 작업 방향은 "예약 자체"에서 끝나지 않고, 예약에 참여하는 사람과 사용자 프로필까지 함께 다루는 흐름으로 확장하는 쪽이다.
- 예약 소유자와 참여자는 분리한다. 화면에서 사람 정보는 더 풍부하게 보여주되, 권한과 `내 예약` 기준은 소유자 중심을 유지한다.

## 현재 중요 흐름
- 로그인 후 메인 타임라인에서 회의실별 예약 현황을 본다.
- 빈 슬롯에서 예약을 생성하고, 상세 모달에서 수정/취소를 진행한다.
- `/me`에서 프로필과 비밀번호를 관리하고, 내가 소유한 예약을 조회한다.
- 예약 생성/수정 시 참여자를 검색해서 추가할 수 있고, 환경변수가 설정돼 있으면 참여자 메일 알림까지 보낸다.

## 꼭 지켜야 하는 원칙
- MVP 핵심인 타임라인 기반 빠른 예약 경험을 해치지 않는다.
- 예약 소유권과 참여자 개념을 섞지 않는다.
- `내 예약`은 계속 "내가 소유한 예약" 기준으로 유지한다.
- 화면 식별성은 강화해도 권한 모델은 더 느슨하게 바꾸지 않는다.
- 운영 환경변수가 없더라도 핵심 예약 흐름은 가능한 범위에서 계속 동작해야 한다.
- 중요한 방향 전환, 정책 변경, DB/API 계약 변경은 `docs/change-log.md`에 남긴다.
- 각 push마다 목표, 진행한 것, 남은 것, 작업 원칙은 `docs/push-journal.md`에 남긴다.
