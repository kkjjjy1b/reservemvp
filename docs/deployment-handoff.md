# Deployment Handoff

배포 전후에 PM, 개발자, 운영 담당자가 같은 기준으로 확인할 수 있도록 만든 handoff 문서다.

## 1. 현재 제품 상태
- MVP 핵심 기능 구현 완료
- 로그인 / 로그아웃 / 자동 로그인 동작
- 일간 타임라인 조회 동작
- 빈 슬롯 기반 예약 생성 동작
- 예약 상세 조회 / 수정 / 취소 동작
- 내 정보 조회 / 프로필 수정 / 비밀번호 변경 동작
- 내 예약 목록 조회 동작

## 2. 현재 기술 스택
- `Next.js` App Router
- `TypeScript`
- `Tailwind CSS`
- `Prisma`
- `PostgreSQL`
- 자체 로그인 + `HTTP-only` 쿠키 세션

## 3. 현재 기본 데이터
### 회의실
- `와이낫` / `20명`
- `두잇` / `10명`
- `쏘왓` / `5명`

### 테스트 사용자
- `user-a@company.com` / `Welcome123!` / `김민지`
- `user-b@company.com` / `Welcome123!` / `박준호`
- `user-c@company.com` / `Welcome123!` / `이서연`
- `user-d@company.com` / `Welcome123!` / `최도윤`

## 4. 배포 전 개발자 작업
1. Git 배포용 저장소를 `project-root` 기준으로 연결한다.
2. 운영 PostgreSQL 인스턴스를 준비한다.
3. 운영 환경변수를 설정한다.
4. 운영 DB에 Prisma migration을 적용한다.
5. 운영용 초기 회의실 / 사용자 데이터를 입력한다.
6. `npm run build`로 프로덕션 빌드를 확인한다.
7. 배포 후 운영 URL에서 수동 검수를 진행한다.

## 4-1. Git 연결 상태
- `project-root`는 2026-03-13에 독립 Git 저장소로 초기화되었다.
- 아직 원격 저장소 URL은 연결되지 않았다.
- 배포 전에 최소 1개의 기본 브랜치(`main` 권장)와 원격 `origin`을 연결해야 한다.
- 상위 폴더 `/Users/jaydenkim/Desktop/Codex/reservMVP`의 Git과 분리해서, 배포 단위는 `project-root`만 사용한다.

## 5. 필수 환경변수
- `DATABASE_URL`

추가 운영값이 생기면 이 섹션에 확장한다.

## 5-1. 런타임 기준
- 권장 Node.js 버전: `22.14.0`
- 권장 npm 버전대: `10.x`
- 배포 플랫폼의 런타임 버전도 로컬 기준과 맞춰야 한다.

## 6. 배포 전 체크리스트
### 기술 체크
- `project-root` 원격 저장소 연결 완료
- 배포 대상 브랜치 확정
- 운영 DB 접속 가능
- migration 적용 완료
- 기본 회의실 데이터 입력 완료
- 기본 사용자 데이터 입력 완료
- `npm run build` 통과

### 기능 체크
- 로그인 가능
- 자동 로그인 가능
- 타임라인 조회 가능
- 예약 생성 가능
- 예약 수정 가능
- 예약 취소 가능
- 내 정보 수정 가능
- 내 예약 목록 조회 가능

### 정책 체크
- 예약 가능 시간 `06:00 ~ 24:00`
- 동일 회의실 시간 충돌 차단
- 끝 시각과 다음 예약 시작 시각 동일 시 허용
- 시작 시간이 지난 예약은 수정 불가
- 취소는 가능

## 7. 배포 후 첫 확인 항목
- 실제 운영 도메인에서 로그인 가능 여부
- 쿠키 세션 유지 여부
- DB에 예약 데이터 정상 저장 여부
- 회의실 이름 / 인원 노출 정상 여부
- 모바일 화면에서 기본 사용 가능 여부

## 8. 운영 중 이슈가 생기면 먼저 볼 것
- 서버 로그
- DB 연결 상태
- 최근 배포 이력
- 최근 migration 적용 여부

## 9. 권장 다음 스레드
- 배포 전용 스레드: 운영 DB, 환경변수, 배포 절차 진행
- 배포 후 수정 스레드: 운영 이슈, 버그 수정, 개선 작업 진행
