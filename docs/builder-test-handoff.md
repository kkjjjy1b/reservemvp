# Builder Test Handoff

## 1. 목적
- 이 문서는 Reviewer/QA 관점에서 현재 구현을 `requirements.md`와 `spec.md` 기준으로 점검한 결과를 정리한 것이다.
- Builder는 이 문서를 기준으로 로컬 `npm` 환경에서 실제 실행 검증을 수행하고, 필요한 수정 후 최종 결과를 같은 문서에 이어서 정리한다.

## 2. Reviewer 점검 범위
- 요구사항 누락 여부
- 예약 생성 흐름 이상 여부
- 예약 수정 흐름 이상 여부
- 예약 취소 흐름 이상 여부
- 내 예약 / 남의 예약 권한 분기 정확성
- 시작 전까지만 수정 가능 정책 반영 여부
- 취소는 언제든 가능 정책 반영 여부
- 종료 시간 선택 단계의 즉시 안내 구현 여부
- 예약 충돌 방지 로직과 UI 동작의 일치 여부
- 반응형 UI 문제 가능성
- 타임라인 UX 문제 가능성
- MVP 범위 이탈 여부

## 3. Reviewer 검토 결과 요약

### 3.1 필수 수정 사항
1. 메인 타임라인과 예약 생성 모달이 `과거 날짜`, `오늘 기준 현재 시각 이전 시작 시간`을 UI 단계에서 예약 불가로 처리하지 않는다.
   - 현재는 서버가 저장 시점에만 거절한다.
   - 요구사항상 UI에서도 예약 불가 상태가 직관적으로 보이고, 종료 시간 선택 단계에서 즉시 안내되어야 한다.

### 3.2 선택 개선 사항
1. 예약 수정 모달은 시간 변경 시 충돌/불가 여부를 즉시 안내하지 않는다.
2. 내 예약 목록에는 `cancelled` 예약도 보이는데, 해당 행 클릭 시 상세 API가 404를 반환할 가능성이 있다.
3. 모바일/터치 환경에서 빈 슬롯의 클릭 가능성이 hover 중심으로 표현되어 상호작용 힌트가 약하다.

## 4. Reviewer가 코드 기준으로 확인한 내용

### 4.1 요구사항과 대체로 일치하는 부분
- 로그인, 메인 타임라인, 내 정보 화면 존재
- 빈 슬롯 클릭 기반 예약 생성
- 예약 블록 클릭 기반 상세 조회
- 내 예약만 수정/취소 가능
- 남의 예약은 조회만 가능
- 시작 전까지만 수정 가능
- 취소는 시작 시간과 무관하게 가능
- 서버 충돌 검증 존재
- DB 수준 중복 방지 제약 존재
- 예약 블록 길이가 시간 길이에 비례하도록 렌더링됨
- 내 예약 목록에 회의실명, 날짜, 시작/종료, 목적, 상태 표시
- MVP 범위 밖 기능 추가 없음

### 4.2 Reviewer가 코드 기준으로 확인한 핵심 리스크
- 타임라인 빈 슬롯은 이미 예약된 슬롯만 막고 있으며, 시간 정책 기반 예약 불가 슬롯은 막지 않음
- 생성 모달의 즉시 안내는 충돌 여부만 반영하며, 시간 정책 위반은 반영하지 않음
- 수정 모달은 즉시 충돌 안내가 없음
- `cancelled` 예약의 내 예약 목록 후속 동작이 현재 UX 설명과 맞지 않을 수 있음

## 5. Reviewer가 실제 실행하지 못한 이유
- 현재 Reviewer 실행 환경에는 `node`, `npm`이 없어 `build`, `lint`, `dev server`, 브라우저 기반 상호작용 테스트를 수행하지 못했다.
- 따라서 아래 항목은 Builder가 실제 실행 검증으로 확정해야 한다.
  - 반응형 UI 실제 동작
  - hydration/runtime 오류 여부
  - 시간대 처리의 실제 런타임 결과
  - 모달 열기/닫기/상태 전이 중 에러 여부
  - 모바일에서 스크롤/터치 상호작용 품질

12. 위 10, 11번 상황에서 생성 모달이 열리지 않거나, 열리더라도 즉시 예약 불가 안내가 보이는지 확인

15. 다른 회의실 같은 시간대 중복 예약이 허용되는지 확인
16. 같은 사용자의 다른 회의실 동시 예약이 허용되는지 확인
17. 예약 생성 성공 시 타임라인에 즉시 반영되는지 확인
18. 내 예약은 시작 전까지 수정 가능한지 확인
19. 내 예약은 시작 시간이 지난 뒤 수정 불가 메시지가 정확히 보이는지 확인
20. 수정 시 회의실 변경이 불가능한지 확인
21. 수정 시 충돌 예약으로 바꾸면 서버가 거절하는지 확인
22. 가능하다면 수정 모달에서도 시간 변경 시 즉시 충돌 안내가 필요한지 UX 확인
23. 내 예약은 시작 시간이 지나도 취소 가능한지 확인
24. 취소 버튼 클릭 시 확인 모달이 한 번 더 보이는지 확인
25. 취소 확정 시 타임라인에서 즉시 사라지는지 확인
26. 내 정보 화면에서 내 예약 목록 컬럼이 요구사항과 일치하는지 확인
27. `cancelled` 예약이 내 예약 목록에 표시될 때 클릭 동작이 자연스러운지 확인
28. 모바일 폭에서 타임라인 가로 스크롤이 동작하는지 확인
29. 모바일에서 슬롯 클릭 후 생성 모달 입력이 무리 없는지 확인
